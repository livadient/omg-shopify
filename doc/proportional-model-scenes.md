# Proportional Model Scenes (gpt-image-2 image-edit)

Generates 8 photoreal model scenes per tee — man+woman × close+full ×
front+back — with a proportional-size print (~40-50% of tee width)
that reads as a real product photoshoot. Originated as the Astous
Eurovision launch (2026-05-05/06) and generalised on 2026-05-06 to
run for every Mango approval.

## The 8 scenes

| Label | Gender | Framing | Placement |
|---|---|---|---|
| `01_female_front_close` | female | close | front |
| `02_female_front_full` | female | full body | front |
| `03_male_front_close` | male | close | front |
| `04_male_front_full` | male | full body | front |
| `01_female_back_close` | female | close | back |
| `02_female_back_full` | female | full body | back |
| `03_male_back_close` | male | close | back |
| `04_male_back_full` | male | full body | back |

## Pipeline

1. **Design PNG** — passed as the primary `image=` reference. Forces
   gpt-image-2 to copy the artwork pixel-faithfully (no DALL-E spelling
   drift).
2. **Anchor PNG** — `04_hanger_back.png` from the same product (already
   generated by `compose_marketing_scenes`). Passed as the SECOND
   reference image. gpt-image-2 inherits the proportional print SIZE
   from the anchor instead of stretching the artwork edge-to-edge across
   the torso.
3. **Prompts** — generic Mediterranean lifestyle photography, no
   tee-specific copy. Each prompt explicitly demands "moderate-size print
   ~40-45% of tee width, generous fabric margin all around".
4. **Concurrency** — Semaphore=2. gpt-image-2 high-quality is ~3 min/call;
   8 in parallel hits OpenAI's per-org rate limit.
5. **Idempotent** — if `<label>.png` already exists in `out_dir`, that
   scene is skipped on re-run.

## Cost

- gpt-image-2 high-quality image-edit: ~$0.19/call
- Per tee: 8 × $0.19 = **~$1.50**
- Mango approves 1-2 tees/week → ~$3/week recurring

## When it runs

### Automatic (Option A — wired 2026-05-06)
Every Mango approval. After `compose_marketing_scenes` produces its 6
paste-pipeline scenes (incl. `04_hanger_back.png`), the new pipeline
generates the 8 photoreal scenes using that hanger as the anchor.
Total: 14 scenes per approved tee uploaded to Shopify.

Code path:
- `app/agents/eurovision_scenes.py` — generic 8-scene generator
- `app/agents/design_creator.py::_generate_marketing_scenes` — wires it
  into the approval flow after `compose_marketing_scenes`
- `app/agents/design_creator.py` upload_plan — appends 8 entries with
  proper gender × placement variant linking

### Manual (Option B — retro-fit existing tees)
On-demand script for tees approved before 2026-05-06 (or anywhere the
auto-step failed):

```bash
# Single tee
.venv/Scripts/python -m scripts.regenerate_proportional_scenes <handle>

# Several
.venv/Scripts/python -m scripts.regenerate_proportional_scenes <h1> <h2> <h3>

# ALL active tees (excludes 5 astous tees that use real photography)
.venv/Scripts/python -m scripts.regenerate_proportional_scenes --all

# Dry-run preview
.venv/Scripts/python -m scripts.regenerate_proportional_scenes --all --dry-run
```

The script:
1. Locates the design PNG (`static/design_<handle>.png`)
2. Locates the hanger anchor (`static/proposals/<slug>/04_hanger_back.png`)
3. Generates 8 scenes via `eurovision_scenes.generate_proportional_scenes`
4. Uploads each to Shopify with gender × placement variant linking
5. Skips astous tees (they use real photography, not AI)

Output goes to `static/proposals/<slug>/_proportional/` so it doesn't
collide with the original Mango compose-pipeline outputs.

## Variant linking

Each new scene is uploaded with `variant_ids` matching its gender +
placement. So picking Male/Female × Front/Back on the product page
swaps the gallery to the matching scene, in addition to the existing
TJ Qstomizer mockup + the original closeup_back/front scenes.

```
01_female_front_close → female front variants
02_female_front_full  → female front variants
03_male_front_close   → male front variants
04_male_front_full    → male front variants
01_female_back_close  → female back variants
02_female_back_full   → female back variants
03_male_back_close    → male back variants
04_male_back_full     → male back variants
```

## Tuning knobs

In `app/agents/eurovision_scenes.py`:
- `PHOTOREAL` — the documentary-style preamble. Tweak Mediterranean
  vs other locales here.
- `ARTWORK_SPEC` — the proportional-print spec. Adjust the 40-50%
  range if scenes still come back too big (or too small).
- `SCENES` — the 8 per-scene prompt strings. Mostly generic
  Mediterranean; if a brand pivot makes Cyprus-flavor inappropriate,
  edit the per-scene "old-town alley", "fishing-village door", etc.
- `DEFAULT_CONCURRENCY = 2` — bump if OpenAI raises your rate limits.

## Why gpt-image-2 + anchor (not Pillow paste)

`compose_marketing_scenes` uses Pillow to paste a transparent design
PNG onto a blank-tee photo. That guarantees exact size but produces a
sticker-paste look (flat on top of fabric, no shadow conformance).

gpt-image-2's image-edit pipeline produces photoreal "design printed
on fabric" output — the artwork picks up fabric weave, fold shadows,
and lighting. Trade-off: without an anchor, gpt-image-2 ignores size
instructions and makes the print billboard-sized. The hanger anchor
(at the right proportional size) solves this.

This is documented in `feedback_default_marketing_pipeline.md` —
"For new tees: gpt-image-1 edit (not compose), live mockup ref,
explicit slogan_layout to pin line count" was the old advice; the
new policy is **gpt-image-2 + hanger anchor + proportional spec**.
